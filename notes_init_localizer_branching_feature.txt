Met with Matthew about this CT Localizer Branching again; have a better understanding now:

Currently:
Dicom branching is based on the filenames https://github.com/scanlabmr/scanlab-web/pull/569
Admins can choose a branch per Question - from a single list of branching, as discovered by HD/HD filenames
Branching is applied to both HD (normal scans) and LD (MRI "Add Localizer") scans/reformats
Branching is not used by/for Initial Localizers
Each Initial Localizer is a single images
All initial localizers are potentially shown at once (I think it tried to find the right direction of localizer per SliceView, and will potentially show 3 different initial localizers (one per SliceView) if there are 3 different direction of them) - a Localizer Set - there can be only one Initial Localizer Set (not a concept)

Requested Feature:
In CT, we want the initial localizers to support Branching, also
There should be multiple Initial Localizer Sets. They should support branching, that should be pickable on the CT question type "Localizer Question"
The list of branches for Localizers should be separate from the current Dicom Branches for HD/LD files. So another list of Initial Localizer Set branches that's discovered from file names.
Admin should be able to pick one ILS branch for a Localizer Question, which then has only that Set of Initial Localizers show up (slow reveal) during that CT question (could still be multiple Initial Localizers just like now, but only from that specific branch)
May need a way to (or manually) rename already uploaded Initial Localizers
